h1. Rubytime

Rubytime is a time tracking and invoicing system written in Merb. It is complete rewrite of Rubytime 2.4 which was formerly Rails application.

h2. Changes

 * See CHANGELOG.txt 

h2. Status

**Currently, the master version is very unstable** - we've added lots of new features, but this has also introduced lots of new bugs, which we didn't have time to fix yet. If you're looking for a stable version, try "the 3.1.1 branch":http://github.com/LunarLogicPolska/rubytime/tree/3.1.1.

h2. Setup

h3. Prerequisites

You need to have Ruby version 1.8.6 or 1.8.7 and Rubygems 1.3 installed on your server. If you don't have Rubygems installed get it at "http://www.rubygems.org":http://www.rubygems.org. You also need the Bundler gem.

h3. Get the source

Next get the source code from Rubytime github repository:

bc. git clone git://github.com/LunarLogicPolska/rubytime.git

h3. Install gems

bc. gem bundle

Note: on 64 bit architectures, you may have to set ARCHFLAGS='-arch x86_64' in your shell environment, otherwise some compiled gems (do_mysql) may not work.

h3. Configure

Enter application directory:

bc. cd rubytime

Then:

 * copy config/database.yml.example to config/database.yml, then edit it to match your database configuration
 * copy config/local_config.rb.example to config/local_config.rb, then set application domain and mailer "From" address
 * run @MERB_ENV=production rake db:automigrate@ to create database structure
 * run @MERB_ENV=production rake rubytime:kickstart@ to populate database tables with initial data (admin account, example clients and projects)

h3. Run

Run the application:

bc. merb

Point your browser to url set in config/local/config.rb and login as @admin@ with password @password@
